# Remodex 全局本地 Relay 使用手册

这个 skill 用来处理下面这类场景：

- 升级或重装全局 `remodex` 后，本地 relay 默认逻辑被覆盖
- 想把 Remodex 重新固定到局域网本地 relay
- 想保留“裸 `remodex up` / `remodex restart` 默认走本地 relay”的行为

## 一步到位

以后最省事直接用这个：

```bash
./skills/remodex-global-local-relay/scripts/bootstrap_global_local_relay.sh --hostname 192.168.1.105
```

默认会一次做完：

1. 升级全局 `remodex`
2. 固定全局默认本地 relay
3. 生成 Shadowrocket `DIRECT` 模块
4. 直接打印新的二维码 / pairing code

如果这次不想升级：

```bash
./skills/remodex-global-local-relay/scripts/bootstrap_global_local_relay.sh --hostname 192.168.1.105 --skip-upgrade
```

如果这次不想重新出二维码：

```bash
./skills/remodex-global-local-relay/scripts/bootstrap_global_local_relay.sh --hostname 192.168.1.105 --skip-pair
```

## 最常用

以后直接用这个就行：

```bash
./skills/remodex-global-local-relay/scripts/upgrade_global_remodex.sh --hostname 192.168.1.105
```

作用：

1. 先升级全局 `remodex`
2. 再自动把“全局默认本地 relay”配置重新打回去

## 只切本地，不升级

如果这次没有升级全局 `remodex`，只是想重新固定到本地 relay，直接用：

```bash
./skills/remodex-global-local-relay/scripts/enable_global_local_relay.sh --hostname 192.168.1.105
```

如果还要直接重新出二维码：

```bash
./skills/remodex-global-local-relay/scripts/enable_global_local_relay.sh --hostname 192.168.1.105 --pair
```

## 如果还要重新出二维码

```bash
./skills/remodex-global-local-relay/scripts/upgrade_global_remodex.sh --hostname 192.168.1.105 --pair
```

这个模式会在升级和重打配置完成后，直接输出新的二维码 / pairing code。

## 自动生成 Shadowrocket 模块

如果你想按当前 Remodex 本地 relay 自动生成 Shadowrocket 的 `DIRECT` 模块，直接用：

```bash
./skills/remodex-global-local-relay/scripts/generate_shadowrocket_direct_module.sh
```

作用：

1. 自动读取当前 Remodex 持久化的本地 relay 地址
2. 生成一份对应的 Shadowrocket `.sgmodule`
3. 用于手机和 Mac 两端导入，确保本地 relay 流量走 `DIRECT`

前提：

1. 当前 `remodex status --json` 里的 `relayUrl` 已经是本地 `ws://<LAN_IP>:9000/relay`
2. 如果当前还是 hosted relay，先跑上面的“升级并重打”或“只切本地，不升级”

当前这台机器生成出来的文件示例：

```bash
./skills/remodex-global-local-relay/references/remodex_shadowrocket_direct_192.168.1.105.sgmodule
```

如果你要手动指定地址，也可以用：

```bash
./skills/remodex-global-local-relay/scripts/generate_shadowrocket_direct_module.sh --hostname 192.168.1.105
```

或者：

```bash
./skills/remodex-global-local-relay/scripts/generate_shadowrocket_direct_module.sh --relay-url ws://192.168.1.105:9000/relay
```

如果当前 relay 主机是 `.local` 或 `.ts.net`，脚本会生成 `DOMAIN,<host>,DIRECT` 规则；如果你想要更精确的 `IP-CIDR,192.168.x.x/32`，建议显式传 `--hostname <LAN_IP>`。

## 参数说明

- `--hostname`
  填这台 Mac 当前在局域网里给 iPhone 访问的 IP 或 `.local` 主机名
- `--pair`
  需要立刻重新配对时加上
- `--package-spec`
  升级全局 `remodex` 时指定版本，例如 `remodex@1.3.8`
- `--output`
  生成 Shadowrocket 模块时自定义输出路径
- `--skip-upgrade`
  总控脚本里跳过 `npm install -g`
- `--skip-module`
  总控脚本里跳过 Shadowrocket 模块生成
- `--skip-pair`
  总控脚本里不输出新的二维码 / pairing code

## 什么时候要重新执行

- 换了 Wi-Fi，Mac 的局域网 IP 变了
- 全局 `remodex` 被重新安装或升级了
- 发现 `remodex up` 又开始回到公用 relay
- 需要重新生成一份适配当前局域网地址的 Shadowrocket 模块

## 配对建议

- 第一次从 hosted relay 改回本地 relay 时，建议 iPhone 和 Mac 两端先临时关掉 Shadowrocket
- 完成一次稳定配对、确认本地 relay 正常后，再重新打开 Shadowrocket
- 重新打开后，要把生成的 `.sgmodule` 分别导入手机和 Mac，并放在默认代理规则前面

## 用完怎么确认

可以检查这三项：

```bash
remodex status --json
defaults read com.emanueledipietro.RemodexMenuBar remodex.menuBar.relayOverride
lsof -nP -iTCP:9000 -sTCP:LISTEN
```

看到 `relayUrl` 是 `ws://192.168.1.105:9000/relay`，并且 `9000` 端口在监听，就说明切换成功。

如果是 Shadowrocket 模块，还可以确认生成的 `.sgmodule` 里包含：

```bash
IP-CIDR,192.168.1.105/32,DIRECT,no-resolve
```

然后把这份模块分别导入手机和 Mac 的 Shadowrocket，并放在默认代理规则前面。

## 常见问题

- `remodex up` 又回到公用 relay：
  重新执行 `upgrade_global_remodex.sh --hostname <LAN_IP>`，或者至少执行一次 `enable_global_local_relay.sh --hostname <LAN_IP>`
- `enable_global_local_relay.sh` 报“hostname 不属于当前这台 Mac”：
  说明你给的 `--hostname` 没有解析到本机；改用当前真实局域网 IP，或者能解析回本机的 `.local`
- 生成 Shadowrocket 模块时报“当前 relay 主机不是局域网地址”：
  说明你现在还没真正切到本地 relay；先切本地，再生成
- 模块生成出来只有 `DOMAIN,<host>,DIRECT` 没有 `IP-CIDR,<LAN_IP>/32`：
  说明当前 relay 主机是 `.local` 或 `.ts.net`；如果你想固定成 IP 规则，用 `--hostname <LAN_IP>` 重生一份
